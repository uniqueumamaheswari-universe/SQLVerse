## Platform: Azure SQL Database (Permanent Free Tier)

### **Why Azure SQL Free Tier is Perfect for Learning:**
- **Permanent Free Allowance**: Provides a fixed amount of usage each month, designed to last the life of the course. This includes **100,000 vCore seconds** of serverless database compute and a maximum size of **32 GB of data** per database.
- **Browser-accessible tools**: The Azure Portal's Query Editor allows you to run queries and manage data directly in your browser. Azure Data Studio also has a web version.
- **Enterprise environment**: Learn on real cloud infrastructure used by businesses.
- **Automatic management**: Backups, updates, and security handled by Azure.

### Setup Steps:
1. **Azure Account Setup:**
   - Go to azure.microsoft.com/free
   - Click "Start free" button
   - Sign up with Microsoft account or create new
   - **Credit card required for verification (not charged unless you upgrade)**

2. **Create SQL Database:**
   - In Azure portal, click "Create a resource" (plus icon top-left)
   - Search "SQL Database" and select it
   - Click "Create" button

3. **Basic Configuration:**
   - Subscription: Select your free subscription
   - Resource group: Click "Create new" → Name: sql-learning-rg
   - Database name: mssql-learning-db (or your preferred name)

4. **Server Configuration (Click "Create new" under Server):**
   - Server name: mssql-learning-server-{yourname} (must be globally unique)
   - Location: Choose region nearest to you (e.g., "East US")
   - Authentication method: Select "Use SQL authentication"
   - Server admin login: sqladmin (or your chosen admin)
   - Password: Create strong password and save it securely!
   - Click "OK"

5. **Pricing Tier Selection (CRITICAL STEP - Permanent Free Tier):**
   - Click "Configure database".
   - **IMPORTANT:** Look for the banner that says "Want to try Azure SQL Database for free?" and select the **"Apply offer"** button.
   - You can create up to 10 free databases under this offer per Azure subscription.
   - **DO NOT** select Basic or Standard tiers unless you understand the monthly charge.

6. **Finalize Creation:**
   - Click "Review + create"
   - Review settings, click "Create"
   - Wait 2-5 minutes for deployment

7. **Connect to Your Database:**
   - After deployment, click "Go to resource"
   - Find "Connection strings" in left menu
   - Copy "ADO.NET" connection string
   - Save your password securely!

8. **Verify Auto-Pause (MANDATORY) 🛡️**
   - Navigate to your SQL Database resource in the Azure Portal.
   - Go to the **Compute + Storage** blade.
   - **CRITICAL:** Ensure the option **"Auto-pause the database until next month"** is selected. This prevents charges if you exceed the free monthly limits.
   - **Warning:** Once you choose "Continue using database for additional charges", you cannot revert to the auto-pause option.

**Total Setup Time:** 15-20 minutes

## 💡 **Testing T-SQL Features in SQL Server**

### **Verification and Feature Testing:**

```sql
-- Check SQL Server version and edition
SELECT @@VERSION AS 'SQL Server Version';

-- Test stored procedure creation (T-SQL specific)
CREATE PROCEDURE GetDepartmentSummary
    @DepartmentID INT
AS
BEGIN
    SELECT 
        DepartmentID,
        COUNT(*) AS EmployeeCount,
        AVG(Salary) AS AverageSalary,
        MIN(Salary) AS MinimumSalary,
        MAX(Salary) AS MaximumSalary
    FROM Employees
    WHERE DepartmentID = @DepartmentID
    GROUP BY DepartmentID;
END;
GO

-- Execute the stored procedure
EXEC GetDepartmentSummary @DepartmentID = 1;

-- Test TRY...CATCH error handling (T-SQL specific)
BEGIN TRY
    -- This will cause an error
    SELECT 1/0 AS DivideByZeroTest;
END TRY
BEGIN CATCH
    SELECT 
        ERROR_NUMBER() AS ErrorNumber,
        ERROR_MESSAGE() AS ErrorMessage;
END CATCH;
```

## 🆘 **Troubleshooting & Cost Management**

### **Platform Comparison:**

| Feature | Azure SQL Database (Free Tier) | Local SQL Server (Developer Ed) | SQL Fiddle |
| :--- | :--- | :--- | :--- |
| **Cost** | Free (Permanent Allowance) | Free Developer Edition | Free |
| **Setup Time** | 15-20 minutes | 30-60 minutes | 2 minutes |
| **Browser Access** | Yes (Azure tools) | No | Yes |
| **Enterprise Features** | Full | Full | Limited |
| **Usage** | Limited Monthly Allowance | Unlimited | Unlimited |

### **Cost Management Tips (Even on Free Tier):**
1. **Monitor spending** in Azure Cost Management periodically.
2. The **Auto-Pause feature** you enabled is your primary safeguard against any charges. Once the monthly limits on vCore activity or storage are met, the database will auto-pause until the beginning of the next calendar month.
3. If you run complex queries that exceed the free allowance, your database will pause rather than charge you (as long as you selected the auto-pause option).
4. **Pro Tip:** Disconnect querying tools when you're done using them. Leaving connections open can continue to consume credits by preventing auto-pause.
