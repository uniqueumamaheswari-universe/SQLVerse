## 🛠️ Step 1: Install SQL Server Developer Edition

### **1.1 Download from Official Source**
**DO NOT search Google.** Use this direct Microsoft link to ensure safety:
- **Official Download:** [SQL Server Developer Edition](https://www.microsoft.com/en-us/sql-server/sql-server-downloads)
- Click the **"Download now"** button under **Developer Edition**

### **1.2 Run the Installation**
1. Run the downloaded `SQLServer*.exe` file
2. Choose installation type: Select **"Basic"** (recommended for simplicity)
3. Accept the license terms
4. Choose installation location (default is fine)
5. **CRITICAL: Note the Instance Name**
   - For Basic installation: Instance name is usually `MSSQLSERVER`
   - You'll need this for connections

### **1.3 Configure Authentication (Important!)**
During installation, you'll see an option for **Authentication Mode**:
- **Select "Mixed Mode"** (SQL Server and Windows Authentication)
- **Set a strong SA (System Administrator) password** and save it securely
- This enables connections from Azure Data Studio and other tools

### **1.4 Complete Installation**
Wait for installation to complete (10-20 minutes depending on your system).

## 🛠️ Step 3: Connect to Your Local Server

### **3.1 Determine Your Server Name**
- **For Basic Installation:** `localhost` or `.` (dot)
- **For Named Instance:** `localhost\INSTANCENAME` or `.\INSTANCENAME`
- **If unsure:** Open SQL Server Configuration Manager → SQL Server Services → Look for "SQL Server (INSTANCENAME)"

### **3.2 Connect in Azure Data Studio**
1. Open Azure Data Studio
2. Click **"New Connection"** (plug icon)
3. Enter connection details:
   - **Server:** `localhost` (or your instance name)
   - **Authentication Type:** **SQL Login**
   - **Username:** `sa`
   - **Password:** The SA password you set during installation
4. Click **"Connect"**

### **3.3 Troubleshooting Connection Issues**
- **"Cannot connect to localhost"**: Ensure SQL Server service is running (Services → SQL Server (MSSQLSERVER))
- **"Login failed"**: Verify you're using SQL Login with correct SA password
- **"Server not found"**: Try `.` instead of `localhost`

## 🆘 **Troubleshooting Common Issues**

| Problem | Solution |
| :--- | :--- |
| **"Cannot connect to localhost"** | Open **SQL Server Configuration Manager** → Enable **TCP/IP protocol** → Restart SQL Server service |
| **"Login failed for user 'sa'"** | Enable SA account: `ALTER LOGIN sa ENABLE;` then `ALTER LOGIN sa WITH PASSWORD = 'new_password';` |
| **"File not found" for database scripts** | Verify you downloaded the **`Transact-SQL-MSSQL`** folders (not `T-SQL_Enterprise`) |
| **Out of disk space during install** | Developer Edition needs ~6GB free space; clear temporary files |
| **Azure Data Studio can't connect** | Use **SQL Login** with username `sa` and your password |
